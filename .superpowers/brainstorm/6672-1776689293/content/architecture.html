Architecture Overview

How the pieces fit together

Directory Structure

app/
layout.tsx — root layout with left nav
page.tsx — dashboard (redirects to /dashboard)
dashboard/
page.tsx — project cards list (Server Component)
chat/
page.tsx — AI chat panel (Client Component)
api/chat/
route.ts — streams Anthropic response
components/
NavSidebar.tsx — left nav (Client, for active link state)
ProjectCard.tsx — card with open loops accordion
OpenLoopsPanel.tsx — expandable loops list (Client)
AddLoopForm.tsx — inline add loop form (Client)
QuickActions.tsx — mark worked on + add loop buttons (Client)
ChatPanel.tsx — streaming chat UI (Client)
lib/
supabase.ts — server + client instances
actions.ts — all Server Actions (mutations)
context.ts — builds AI system prompt from project data

Data Flow

INITIAL LOAD
Browser → Next.js Server
↓ Server Component fetches
Supabase (projects + loops)
↓ Renders HTML with data
Hydrates Client Components
MUTATIONS
User clicks action
↓ Client optimistic update
Server Action called
↓ Supabase write
revalidatePath() refreshes
AI CHAT
User sends message
↓ POST /api/chat
Fetch all projects + loops
↓ Stream from Anthropic
ReadableStream → UI

Key Boundaries

SERVER COMPONENTS
dashboard/page.tsx reads Supabase directly — no API route needed for reads
CLIENT COMPONENTS
Any component with onClick, useState, or useEffect — open loops, quick actions, chat
